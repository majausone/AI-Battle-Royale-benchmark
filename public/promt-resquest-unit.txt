# Guide to Create a Unit in Kewo Battle

## 1. Introduction

You must create four files for a complete character in Kewo Battle:

1. A unit file (foo-8472.json)
2. A skill file (skill-foo-8472.js)
3. A skill effect file (seffect-foo-8472.json)
4. A visual effect file (fx-foo-8472.js) (create ONLY 1 FX of your choice and no more)

IMPORTANT: You must generate a random 4-digit ID (example: "8472") and add it to ALL file names and internal IDs, separated by a hyphen.

## 2. File Nomenclature

The correct naming convention is:
- Unit: kewoName-8472.json (WITH hyphen before the ID)
- Skill: skill-name-8472.js (with hyphens)
- Skill effect: seffect-name-8472.json (with hyphens)
- Visual effect: fx-name-8472.js (with hyphens)

## 3. Creating a Unit (kewoName-8472.json)

        "move2": [ /* same matrix size as graphics */ ],
        "attack": [ /* same matrix size as graphics */ ]

    "sounds": {

### 3.1. Graphics Format

Graphics are defined as a 2D array of colors:
Size MUST be between 3x3 and 12x12 (any rectangle in that range).

"graphics": [
    [0, 0, "#FF0000", 0, 0],
    [0, "#FF0000", "#FF0000", "#FF0000", 0],
    [0, "#FF0000", "#000000", "#FF0000", 0],
    ["#8B4513", "#FF0000", "#FF0000", "#FF0000", "#8B4513"],
    [0, "#8B4513", "#8B4513", "#8B4513", 0],
    [0, "#8B4513", 0, "#8B4513", 0],
    [0, "#8B4513", 0, "#8B4513", 0]
]

You must include "animationGraphics" with the same dimensions as "graphics":
- Frame order: graphics (idle/frame 1) -> animationGraphics.move1 (frame 2) -> animationGraphics.move2 (frame 3) loop during movement/idle.
- When the unit attacks, animationGraphics.attack (frame 4) is shown for 1 second.

## 4. Creating a Skill (skill-name-8472.js)

Skills define the special behavior of units. There are 7 types:
- AURA: Affects units in an area
- BUFF: Improves statistics
- DEBUFF: Weakens enemy statistics
- TRANSFORM: Changes appearance and statistics
- SUMMON: Creates new units
- PROJECTILE: Launches special projectiles
- TELEPORT: Allows teleportation

### 4.1. Basic Skill Structure

import { BaseSkill } from '../baseSkill.js';

class NameSkill extends BaseSkill {
    constructor() {
        super();
        this.id = "skill-name-8472";
        this.metadata = {
            name: "Skill Name",
            description: "Description of what it does",
            skillType: "SKILL_TYPE", // AURA, BUFF, DEBUFF, TRANSFORM, SUMMON, PROJECTILE, TELEPORT
            
            // Associated seffect ID
            seffects: "seffect-name-8472",
            
            // Activation system (at least one is necessary)
            trigger: {
                // Some examples - use appropriate ones according to the table:
                onAttack: false,    // When attacking
                onGetAttacked: true, // When being attacked
                onSpawn: false,     // When spawning
                interval: 5000,     // Interval in ms
                chance: 0.5,        // Probability (0-1)
                healthThreshold: 0.3 // % of life to activate
            },
            
            // FX (use allowed ones according to type)
            fx: {
                start: "fx-name-8472",
                continuous: null,
                end: null,
                impact: null
            },
            
            // Sounds
            sounds: {
                start: ["type", frequency, duration, ...parameters], // See section 5
                end: null
            },
            
            // Type-specific properties
            // (properties like projectileConfig, summonProperties, etc. would go here)
        };
    }

    apply(unit) {
        super.apply(unit);
    }
}

const skill = new NameSkill();
export default skill;

## 6. Creating a Skill Effect (seffect-name-8472.json)

The seffects define what happens when a skill is activated. There are 4 types:
- STAT: Modifies a statistic
- MULTI_STAT: Modifies multiple statistics
- AURA: Affects units in an area
- PROJECTILE: Defines the effect of a projectile

### 6.1. Basic Structure of a seffect

{
    "id": "seffect-name-8472",
    "effectType": "EFFECT_TYPE", // STAT, MULTI_STAT, AURA, PROJECTILE
    "duration": 3000, // Duration in ms (-1 for permanent)
    
    // The rest of the properties depend on the effect type
    
    // FX associated with the source
    "sourceFx": {
        "start": "fx-startEffect-8472",
        "continuous": "fx-continuousEffect-8472",
        "end": "fx-endEffect-8472"
    },
    
    // FX associated with the target (not available in AURA)
    "targetFx": {
        "start": "fx-startEffect-8472",
        "continuous": "fx-continuousEffect-8472",
        "end": "fx-endEffect-8472"
    }
}

### 6.2. Types of seffects and Their Specific Properties

#### 6.2.1. STAT
{
    "id": "seffect-stat-8472",
    "effectType": "STAT",
    "duration": 3000,
    "stat": "damage", // "damage" or "speed" (optional if using damageAmount, healAmount or shieldAmount)
    "value": 5, // Value to add (can be negative)
    "shieldAmount": 2, // Amount of damage blocked (optional)
    "damageAmount": 1, // Amount of damage to inflict per pulse (optional)
    "healAmount": 2, // Amount of healing to provide per pulse (optional)
    "pulseInterval": 500, // Interval for damageAmount/healAmount effects (in ms)
    
    "sourceFx": {
        "start": "fx-statStart-8472",
        "continuous": "fx-statContinuous-8472"
    }
}

## 7. Creating a Visual Effect (fx-name-8472.js)

FX define how skills and effects are visualized. There are 7 main types:
- GLOW: Glow around the unit
- PARTICLES: Particles emanating from the unit
- SHIELD: Visual shield
- EXPLOSION: Explosion
- PULSE: Energy pulse
- TRAIL: Trail left by the unit
- ATTACK: Attack animation

### 7.1 Basic Structure of an FX

import baseFx, { EFFECT_TYPES } from '../baseFx.js';

const effectId = "fx-name-8472";

export default async function nameEffect(target, data) {
    // If it's a simple effect, you can use baseFx.createEffect:
    return await baseFx.createEffect({
        id: effectId,
        target,
        type: EFFECT_TYPES.EFFECT_TYPE, // GLOW, PARTICLES, SHIELD, EXPLOSION, PULSE, TRAIL, ATTACK
        duration: 500, // Duration (for temporary effects)
        properties: {
            // Type-specific properties
            color: 'rgba(255, 0, 0, 1)',
            size: 1.2,
            // ... other properties
        }
    });
}

nameEffect.id = effectId;

### 7.2 FX Examples

#### 7.2.1 GLOW
return await baseFx.createEffect({
    id: effectId,
    target,
    type: EFFECT_TYPES.GLOW,

        pulseSpeed: 300, // Pulse speed

    }
});

#### 7.2.2 PARTICLES

        particleCount: 15,

        particleColor: '#32CD32',
        particleLife: 1000,
        particleGravity: 0.1,
        particleShape: 'circle', // 'circle', 'square', 'line'
        emitterShape: 'explosion', // emission shape
        fadeOut: true
    }
});

## 9. Allowed Ranges and Balancing

### 9.1 Ranges for Units
When creating a unit, it's important to stay within the allowed ranges and properly balance the statistics to avoid units that are too powerful or too weak.

Property,Minimum Value,Maximum Value,Notes
cost,80,200,Cost in game resources
life,8,20,Life points
speed,0.8,3.0,Movement speed
damage,0,15,Base damage per attack
attackRange,40,60,For melee units
attackSpeed,500,2000,Time between attacks in ms (lower = faster)
optimalRange,300,400,For ranged units
projectileSpeed,3,6,Projectile speed

### 9.2 Ranges for Effects (seffects)
For skills and effects, these are the recommended ranges to keep the game balanced:

Property,Minimum Value,Maximum Value,Notes
duration,1000,10000,Duration in ms (-1 for permanent)
value (damage),-10,10,Damage modifier
value (speed),-1.5,1.5,Speed modifier
auraRadius,50,200,Aura radius in pixels
damageAmount,1,5,Damage per pulse
healAmount,1,5,Healing per pulse
pulseInterval,500,3000,Interval between pulses
shieldAmount,1,5,Amount of damage blocked

### 9.3 Balancing Tips
If a unit has a very powerful ability, compensate it with weaker base statistics.
If it heals in an area, or damages in an area, take that into account so it's not too powerful.


## 10. RESPONSE FORMAT

You must create an ID that is 4 random numbers and add it to all file names and IDs of things.
Your response must be plain text separating each file only by the comment of its file name, WITHOUT including paths.
Remember to consult the tables in section 8, especially 8.1 and 8.2 to know what things you should and should not include.

You must create {{UNITS_NUMBER}} complete units (with their respective skills, seffects, and ONE FX per unit).

For example, if your ID is "8472", your response should be:

// kewoName-8472.json
{
    "id": "kewoName-8472",
    ...
}
